## 2. 设计原则

### 2.1 核心原则

#### 简洁优先 (Simplicity First)

- ✅ 代码简单易懂
- ✅ 避免过度设计
- ✅ 功能聚焦核心

**例子**:
```typescript
// ✅ 简单直接
const addStroke = (stroke) => {
  set((state) => ({
    strokes: [...state.strokes, stroke]
  }))
}

// ❌ 过度设计
const StrokeManager = {
  addStroke: (stroke, options, metadata, callbacks) => {
    // 复杂的逻辑...
  }
}
```

---

#### 用户至上 (User First)

- ✅ 零门槛使用
- ✅ 快速响应
- ✅ 友好的错误提示

**实现**:
```typescript
// 友好的错误处理
try {
  exportAsPDF()
} catch (error) {
  alert('导出失败，请重试')
  console.error(error) // 开发者能看到详细信息
}
```

---

#### 性能优化 (Performance Matters)

- ✅ 60fps 渲染
- ✅ 低延迟响应
- ✅ 合理的内存使用

**优化措施**:
- 使用 Canvas 而非 SVG（性能更好）
- 使用 perfect-freehand（专业的笔迹算法）
- 使用 Zustand（轻量状态管理）

---

#### 隐私保护 (Privacy First)

- ✅ 数据本地存储
- ✅ 不上传云端
- ✅ 无追踪代码

**实现**:
```typescript
// 所有数据存储在浏览器本地
const saveNote = (note) => {
  localStorage.setItem('note', JSON.stringify(note))
  // 不发送到任何服务器
}
```

### 2.2 架构决策

#### 为什么用 Canvas 不用 SVG？

| 特性 | Canvas | SVG | 选择 |
|------|--------|-----|------|
| **性能** | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ | Canvas |
| **复杂度** | ⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ | Canvas |
| **可访问性** | ⭐⭐ | ⭐⭐⭐⭐⭐ | SVG |
| **缩放** | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ | SVG |

**决策**: 选择 Canvas，因为性能更重要

---

#### 为什么用 Zustand 不用 Redux？

| 特性 | Zustand | Redux | 选择 |
|------|---------|-------|------|
| **学习曲线** | ⭐⭐⭐⭐⭐ | ⭐⭐ | Zustand |
| **代码量** | ⭐⭐⭐⭐⭐ | ⭐⭐ | Zustand |
| **性能** | ⭐⭐⭐⭐ | ⭐⭐⭐⭐ | 平手 |
| **生态** | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ | Redux |

**决策**: 选择 Zustand，因为项目简单，不需要 Redux 的复杂性

---

#### 为什么用 Vite 不用 Webpack？

| 特性 | Vite | Webpack | 选择 |
|------|------|---------|------|
| **启动速度** | ⭐⭐⭐⭐⭐ | ⭐⭐ | Vite |
| **热更新** | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ | Vite |
| **配置复杂度** | ⭐⭐⭐⭐⭐ | ⭐⭐ | Vite |
| **生态** | ⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ | Webpack |

**决策**: 选择 Vite，因为开发体验更好

## 4. 核心架构

### 4.1 组件架构

```
App (根组件)
│
├── Canvas (画布组件)
│   ├── 处理用户输入
│   ├── 渲染笔迹
│   └── 响应式调整
│
├── Toolbar (工具栏组件)
│   ├── 工具选择
│   ├── 颜色选择
│   └── 粗细调整
│
└── SaveDialog (保存对话框)
    ├── 格式选择
    ├── 导出功能
    └── 加载状态
```

### 4.2 状态架构

```
useAppStore (全局状态)
│
├── 数据状态
│   ├── strokes: Stroke[]
│   └── currentStroke: Stroke | null
│
├── 工具状态
│   ├── tool: 'pen' | 'eraser'
│   ├── color: string
│   └── size: number
│
└── 操作方法
    ├── addStroke()
    ├── updateCurrentStroke()
    ├── startStroke()
    ├── clearStrokes()
    ├── setTool()
    ├── setColor()
    ├── setSize()
    ├── undo()
    └── redo()
```

### 4.3 渲染架构

```
用户输入
  ↓
PointerEvent
  ↓
Canvas 组件事件处理
  ↓
调用 Store 方法
  ↓
更新状态
  ↓
React 检测状态变化
  ↓
触发重新渲染
  ↓
Canvas 清空
  ↓
遍历所有笔迹
  ↓
使用 perfect-freehand 生成路径
  ↓
绘制到 Canvas
  ↓
完成渲染
```

## 5. 数据流

### 5.1 单向数据流

```
用户操作 → Store 方法 → 状态更新 → 组件重新渲染 → UI 更新
```

**例子**:

```typescript
// 1. 用户点击
<button onClick={() => setTool('pen')} />

// 2. 调用 Store 方法
const setTool = (tool) => set({ tool })

// 3. 状态更新
state.tool = 'pen'

// 4. 组件重新渲染
const { tool } = useAppStore()

// 5. UI 更新
<button className={tool === 'pen' ? 'active' : ''} />
```

### 5.2 笔迹数据流

```
用户鼠标按下
  ↓
创建新笔迹对象
{
  id: '123',
  points: [[x, y, pressure]],
  color: '#000',
  size: 4,
  tool: 'pen'
}
  ↓
添加到 currentStroke
  ↓
用户鼠标移动
  ↓
更新 points 数组
  ↓
用户鼠标松开
  ↓
添加到 strokes 数组
  ↓
渲染所有笔迹
```

## 6. 性能考虑

### 6.1 渲染性能

#### 挑战

- 大量笔迹（1000+）
- 实时渲染（60fps）
- 平滑曲线（复杂计算）

#### 解决方案

**1. 使用 Canvas API**
```typescript
// Canvas 比 SVG 快 10 倍以上
ctx.beginPath()
ctx.moveTo(x, y)
ctx.lineTo(x2, y2)
ctx.stroke()
```

**2. 使用 perfect-freehand**
```typescript
// 专业的笔迹算法，性能优秀
const pathData = getStroke(points, options)
```

**3. 避免不必要的重新渲染**
```typescript
// 使用 React.memo
const Canvas = React.memo(({ strokes }) => {
  // 只在 strokes 变化时重新渲染
})
```

### 6.2 内存管理

#### 挑战

- 大量笔迹数据
- 长时间使用
- 移动设备内存有限

#### 解决方案

**1. 数据压缩**
```typescript
// 只存储必要的信息
interface Stroke {
  id: string
  points: number[][]  // [x, y, pressure]
  color: string
  size: number
}
```

**2. 分页加载**（未来）
```typescript
// 只渲染可见区域的笔迹
const visibleStrokes = strokes.filter(isVisible)
```

**3. 自动清理**（未来）
```typescript
// 清理超过一定时间的笔迹
if (strokes.length > MAX_STROKES) {
  strokes.shift() // 移除最早的笔迹
}
```

### 6.3 构建优化

#### 代码分割

```typescript
// vite.config.ts
build: {
  rollupOptions: {
    output: {
      manualChunks: {
        vendor: ['react', 'react-dom'],
        utils: ['perfect-freehand', 'jspdf']
      }
    }
  }

```

#### Tree Shaking

```typescript

// ❌ 差的：导入整个库
import * as perfectFreehand from 'perfect-freehand'
```

## 7. 安全考虑

### 7.1 数据安全

#### 本地存储

```typescript
// 数据存储在浏览器本地
localStorage.setItem('notes', JSON.stringify(notes))

// 不发送到任何服务器
// 没有后端
// 没有数据库
```

#### 导出安全

```typescript
// 导出是纯前端操作
const exportAsPNG = () => {
  canvas.toBlob((blob) => {
    saveAs(blob, 'note.png')
    // 不经过服务器
  })
}
```

```typescript
// ✅ 好的：使用 React 的转义

<div dangerouslySetInnerHTML={{ __html: userInput }} />
```

## 8. 扩展性

### 8.1 添加新功能

#### 步骤

1. **创建组件** - `src/components/NewFeature.tsx`
2. **添加状态** - `src/store/useAppStore.ts`
3. **引入使用** - `src/App.tsx`
4. **测试** - 确保功能正常

#### 例子：添加深色模式

```typescript
// 1. 添加状态
const darkMode = create((set) => ({
  darkMode: false,
  toggleDarkMode: () => set((state) => ({ darkMode: !state.darkMode }))
}))

// 2. 创建组件
const DarkModeToggle = () => {
  const { darkMode, toggleDarkMode } = useDarkModeStore()
  return <button onClick={toggleDarkMode}>🌙</button>
}

// 3. 在 App 中使用
<App>
  <DarkModeToggle />
</App>
```
